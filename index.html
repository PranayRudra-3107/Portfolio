Process

How I design
from problem to product.

My process isn't linear — it's a structured loop that adapts to the project's complexity and the team's constraints.

01

Discover

Stakeholder interviews, competitor analysis, existing data review. I frame the problem in terms of user jobs-to-be-done before any sketching happens.

User Interviews Heuristic Evaluation Affinity Mapping
02

Define

Synthesise findings into a clear problem statement, personas, and measurable success criteria. This is the contract between design and the rest of the team.

Personas User Journey Maps HMW Statements
03

Design

Low-fi sketches to high-fi Figma prototypes, grounded in the design system. I test early and often — with real users when possible, with heuristics when not.

Figma Component Libraries Usability Testing
04

Deliver

Developer-ready specs, annotated components, and implementation support. My engineering background means I can close the gap between design intent and shipped product.

Design Tokens Angular / TypeScript Accessibility Audit
05

Measure

Post-launch: task-completion rates, error rates, qualitative feedback, analytics. Data informs the next iteration rather than just validating the last one.

A/B Testing Analytics Retrospectives
